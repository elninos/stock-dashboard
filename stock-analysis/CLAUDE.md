# CLAUDE.md — stock-analysis 공통 규칙

## 작업 폴더 고정
- 모든 분석 작업의 루트는 `stock-analysis/` 폴더로 고정한다.
- 상대 경로는 이 폴더를 기준으로 해석한다.

---

## 데이터 우선순위 (반드시 준수)

1. **CLAUDE.md + ARCHITECTURE.md + CODEBASE.md** — 분석 시작 전 반드시 읽는다.
2. **profiles/{ticker}.md** — 종목별 보정 규칙·예외·맥락을 가장 먼저 읽는다.
3. **data/kis_cache/** — KIS API 캐시 데이터를 로컬 DB 1순위로 사용한다.
4. **pipelines/fetch_daily.py 결과** — 캐시 미스 시 fetch 실행 결과를 2순위로 사용한다.
5. **웹 검색** — 위 1~4에서 데이터가 부족하거나 최신 뉴스·이벤트 확인이 필요할 때만 보조 사용한다.

> ⚠️ 웹 검색을 먼저 실행하지 말 것. 로컬 캐시를 먼저 확인 후 부족분에만 사용한다.

## 분석 공통 규칙

### 종목 유형 분류
- **테마주**: 정책·트렌드 수혜 중심, 실적 연동 약함
- **실적주**: EPS·영업이익 성장 중심
- **수급주**: 외인·기관 수급 중심
- **이벤트주**: M&A·계약·공시 중심 단기 급등
- **복합형**: 위 2가지 이상 혼재

### 기간 3층 분석 구조
| 레이어 | 기간 | 목적 |
|--------|------|------|
| 단기 | 1M | 최근 수급·모멘텀 확인 |
| 중기 | 6M | 추세 형성 여부 판단 |
| 장기 | 1Y | 구조적 상승 논리 검증 |

### 상승 원인 분류
- **유효**: 현재도 작동 중인 상승 논리
- **약화**: 일부 해소됐거나 시장에 선반영된 논리
- **소멸**: 이미 무너진 논리 (보유 근거 재검토 필요)

### 조정/하락 이벤트 판정 기준
- 5% 이상 단기 하락 → 이벤트로 기록
- 원인 분류: 차익실현 / 외부 매크로 / 수급 이탈 / 논리 훼손
- 논리 훼손 판정 시 경고 등급 상향

### 경고 Top 5 기준
1. 상승 논리가 가장 먼저 무너질 선행 지표
2. 수급 이탈 신호 (외인·기관 동반 매도)
3. 과도한 단기 급등 후 거래량 급감
4. 핵심 이벤트 소멸 또는 취소
5. 업종 전체 하락 전환 신호

## 분석 6개 섹션 (보고서 필수 포함)

### 1. 기술적 분석
- 이동평균선(5/20/60/120일) 배열 및 정배열 여부
- RSI(14일) 과매수/과매도 판정 기준: 70 이상 과매수, 30 이하 과매도
- MACD 시그널 크로스 확인
- 볼린저밴드 위치 (상단 돌파 / 중단 지지 / 하단 이탈)
- 거래량 이동평균 대비 당일 거래량 비율 (급증 기준: 5일 평균의 200% 이상)
- 지지선/저항선 주요 레벨 명시
- 캐시: `data/kis_cache/{ticker}_ohlcv.csv` + `{ticker}_technical.json`

### 2. 수급 분석
- 외국인 순매수/순매도 추이 (최근 20거래일)
- 기관 순매수/순매도 추이 (최근 20거래일)
- 외인+기관 동반 매수/매도 여부 판단
- 개인 수급 방향 (역추세 매매 여부 확인)
- 누적 순매수 방향 전환 시점 표시
- 캐시: `data/kis_cache/{ticker}_supply.json`

### 3. 공매도 잔고/비율
- 공매도 잔고 추이 (최근 20거래일)
- 공매도 비율 (당일 거래량 대비 %)
- 공매도 잔고 비율 (상장주식수 대비 %)
- 공매도 급증 시 경고 (전일 대비 20% 이상 증가)
- 대차잔고 변화 방향 확인
- 캐시: `data/kis_cache/{ticker}_short.json`

### 4. 시장환경 / 상대강도
- KOSPI/KOSDAQ 대비 상대강도(RS) 수치
- 업종지수 대비 상대강도
- 최근 1M 시장 대비 초과수익/부진 여부
- 동일 업종 내 동반 상승 또는 독주 여부
- 매크로 환경 요약 (금리·환율·지수 방향)
- 캐시: `data/kis_cache/{ticker}_market.json` (없으면 웹 검색 보완)

### 5. 거래원 / 기관·외국인 흐름
- 주요 거래원 순위 및 순매수 금액 (상위 5개)
- 외국계 창구 집중 여부
- 기관 창구별 매수/매도 패턴
- 프로그램 매매 비중 확인
- 캐시: `data/kis_cache/{ticker}_broker.json`

### 6. 재무 / 실적 요약
- 최근 4개 분기 매출·영업이익·순이익 추이
- YoY / QoQ 성장률
- PER / PBR / ROE (현재 vs. 업종 평균)
- 다음 실적 발표 일정 (있으면 명시)
- 어닝 서프라이즈/쇼크 이력
- 캐시: `data/kis_cache/{ticker}_financial.json`

> 보조 캐시 (분석 참조용): `{ticker}_info.json` (종목 기본정보), `{ticker}_news.json` (뉴스)

## 보고서 저장 규칙
- 저장 위치: `analysis/reports/`
- 파일명: `YYYYMMDD_{종목명}_{티커}.md`
- JSON 부속: `YYYYMMDD_{종목명}_{티커}.json`
- HTML 뷰어: `YYYYMMDD_{종목명}_{티커}_viewer.html`
- 동일 일자 중복 시 `_02`, `_03` 순번 추가
- 보고서는 위 6개 섹션을 모두 포함해야 한다
- 데이터가 없는 항목은 단순 "데이터 없음" 대신 아래 3가지를 함께 기록한다:
  - 캐시 미스 여부: `data/kis_cache/{ticker}_*.json/csv` 파일 존재 및 유효기간 초과 여부
  - fetch 필요 여부: `pipelines/fetch_daily.py --ticker {ticker} --type {종류}` 실행 필요 여부
  - 웹 검색 보완 여부: fetch 실패 또는 실시간 데이터 필요 시 웹 검색 대체 가능 여부

## 판정 문구 기준
- **매수 유효**: 핵심 상승 논리 2개 이상 유효, 경고 없음
- **관망**: 논리 일부 약화 또는 경고 1~2개
- **비중 축소 검토**: 논리 소멸 또는 경고 3개 이상
- **손절 검토**: 논리 전부 소멸, 수급 이탈 확인
